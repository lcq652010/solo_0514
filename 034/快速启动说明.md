# 电影院管理系统 - 快速启动指南

## 系统功能概览

### ✅ 已实现的核心功能

1. **影片 + 场次 + 状态多条件组合筛选**
   - 影片：按名称、类型、导演、是否上映、评分范围筛选
   - 排期：按影片、影厅、日期范围、票价范围、场次状态筛选
   - 订单：按订单号、状态（多状态组合）、影片、影厅、顾客信息、金额范围筛选

2. **检票后状态自动更新**
   - 检票成功后自动将订单状态从"待检票"更新为"已检票"
   - 自动记录检票时间和检票员
   - 防止重复检票、已退票订单检票、已结束场次检票
   - 仅检票员/管理员有权限

3. **退票后座位自动释放**
   - 使用数据库事务确保数据一致性
   - 退票成功后自动释放关联的座位资源
   - 自动记录退票时间和操作人
   - 仅场次开始前允许退票
   - 仅售票员/管理员有权限

4. **三级权限隔离**
   - **管理员**：所有权限，包括用户管理、影片/影厅/排期管理
   - **售票员**：查看数据、创建订单、退票、完结订单、查看统计
   - **检票员**：查看数据、执行检票操作

## 快速启动步骤

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置数据库

确保MySQL已启动，创建数据库：
```sql
CREATE DATABASE cinema_db CHARACTER SET utf8mb4;
```

修改 `cinema_system/settings.py` 中的数据库密码。

### 3. 执行数据库迁移

```bash
python manage.py makemigrations
python manage.py migrate
```

### 4. 初始化用户组

```bash
python manage.py init_cinema_groups
```

### 5. 创建超级管理员

```bash
python manage.py createsuperuser
```

### 6. 启动服务

```bash
python manage.py runserver
```

## 权限验证测试流程

### 测试1：管理员权限

1. 使用超级管理员账号登录后台：http://127.0.0.1:8000/admin/
2. 创建三个测试用户：admin_user, seller_user, checker_user
3. 将三个用户分别添加到：管理员、售票员、检票员用户组

### 测试2：售票员权限测试

1. 使用API登录获取token：
```bash
POST http://127.0.0.1:8000/api/auth/login/
{
    "username": "seller_user",
    "password": "your_password"
}
```

2. 测试：查看影片列表（应该成功）
3. 测试：创建订单（应该成功）

## 检票与退票测试流程

1. **创建订单**（售票员权限）：
   ```json
   POST /api/orders/
   {
       "schedule": 1,
       "seat_ids": [1, 2],
       "customer_name": "张三",
       "customer_phone": "13800138000"
   }
   ```

2. **检票**（检票员权限）：
   ```json
   POST /api/check-ticket/
   {
       "order_no": "TK20240101120000ABC12345"
   }
   ```
   ✅ 订单状态自动更新为"已检票"，记录检票员和时间

3. **退票**（售票员权限）：
   ```json
   POST /api/orders/1/refund/
   ```
   ✅ 订单状态更新为"已退票"，座位自动释放
