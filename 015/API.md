# 银行智慧柜员机运维管理系统 API 文档

## 运行说明

1. 安装依赖：
```bash
pip install -r requirements.txt
```

2. 启动服务：
```bash
python app.py
```

3. 服务地址：http://localhost:5000

---

## 常量定义

### 设备状态
- 正常、故障、维修中、已修复

### 工单状态
- 待处理、处理中、已处理、已完成

### 故障类型
- 硬件故障、软件故障、网络故障、电源故障、打印机故障、读卡器故障、其他

### 优先级
- 低、一般、高、紧急（按金融业务影响程度）

### 网点区域
- 天河区、越秀区、海珠区、荔湾区、白云区、番禺区

## API 接口列表

### 1. 设备管理

#### 1.1 添加设备
- **接口**: `POST /api/devices`
- **请求体**:
```json
{
  "device_code": "ATM001",
  "device_name": "智慧柜员机1号",
  "device_model": "DT-7000",
  "system_version": "V2.3.1",
  "enable_date": "2024-01-15",
  "area": "天河区",
  "location": "天河路1号营业厅",
  "status": "正常"
}
```

#### 1.2 查询设备列表
- **接口**: `GET /api/devices`
- **查询参数**:
  - `status` (可选，按状态筛选)
  - `area` (可选，按区域筛选)
  - `keyword` (可选，关键词搜索)
- **响应说明**: 每个设备包含待处理工单数、处理中工单数

#### 1.3 查询单个设备（设备工单联动）
- **接口**: `GET /api/devices/{device_id}`
- **响应说明**: 返回设备详情 + 最近10条工单 + 最近20条维修日志

#### 1.4 更新设备
- **接口**: `PUT /api/devices/{device_id}`

#### 1.5 删除设备
- **接口**: `DELETE /api/devices/{device_id}`

---

### 2. 工单管理

#### 2.1 创建报修工单
- **接口**: `POST /api/workorders`
- **请求体**:
```json
{
  "device_id": 1,
  "fault_type": "硬件故障",
  "fault_description": "触摸屏无响应，无法操作",
  "priority": "紧急",
  "reporter": "张三"
}
```

#### 2.2 查询工单列表（多维度筛选 + 优先级排序）
- **接口**: `GET /api/workorders`
- **查询参数**:
  - `status` (可选，按状态筛选)
  - `fault_type` (可选，按故障类型筛选)
  - `priority` (可选，按优先级筛选)
  - `area` (可选，按网点区域筛选)
  - `keyword` (可选，关键词搜索)
- **响应说明**:
  - 自动按优先级从高到低排序
  - 包含优先级颜色字段，用于前端高亮显示

#### 2.3 查询单个工单（含维修日志）
- **接口**: `GET /api/workorders/{order_id}`
- **响应说明**: 返回工单详情 + 完整维修日志

#### 2.4 开始处理工单
- **接口**: `PUT /api/workorders/{order_id}/start`
- **请求体**:
```json
{
  "handle_user": "李四",
  "content": "到达现场，开始检查设备"
}
```

#### 2.5 处理工单
- **接口**: `PUT /api/workorders/{order_id}/handle`
- **请求体**:
```json
{
  "handle_user": "李四",
  "handle_result": "更换触摸屏模块，测试正常"
}
```

#### 2.6 完成维修
- **接口**: `PUT /api/workorders/{order_id}/complete`

---

### 3. 运维记录

#### 3.1 查询运维记录
- **接口**: `GET /api/maintenance`
- **查询参数**: `device_id` (可选，按设备筛选)

#### 3.2 添加运维记录
- **接口**: `POST /api/maintenance`

#### 3.3 查询维修日志
- **接口**: `GET /api/maintenance/logs`
- **查询参数**:
  - `device_id` (可选，按设备筛选)
  - `work_order_id` (可选，按工单筛选)

---

### 4. 数据统计

#### 4.1 仪表盘数据
- **接口**: `GET /api/dashboard`
- **返回**:
  - 设备统计（总数、各状态数量、可用率）
  - 工单统计（总数、各状态数量、紧急/高优先级待处理）
  - 故障类型分布统计
  - 各区域设备/故障统计

#### 4.2 设备可用率统计
- **接口**: `GET /api/statistics/availability`
- **查询参数**: `area` (可选，按区域筛选)
- **返回**:
  - 总设备数、正常设备数、故障设备数
  - 整体可用率
  - 各区域可用率详情

---

### 5. 系统配置

#### 5.1 获取配置常量
- **接口**: `GET /api/config`
- **返回**: 故障类型列表、优先级列表、设备状态列表、工单状态列表、区域列表、优先级颜色映射

---

## 统一响应格式

所有接口返回统一格式：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

- `code`: 200 成功，400 参数错误，404 资源不存在，500 服务器错误

## 主要功能特性

### 1. 设备工单联动
- 设备详情页自动关联显示该设备的所有工单
- 工单状态变化自动同步更新设备状态
- 创建/处理/完工单时自动记录维修日志

### 2. 优先级高亮排序
- 工单列表按优先级从高到低自动排序
- 每个优先级配有对应的颜色值（绿色/蓝色/橙色/红色）
- 紧急和高优先级工单在列表中置顶显示

### 3. 多维度筛选
- 支持按设备状态、区域、关键词筛选设备
- 支持按工单状态、故障类型、优先级、区域、关键词筛选工单

### 4. 完整维修日志记录
- 工单全流程日志记录：上报 → 开始处理 → 处理记录 → 维修完成
- 每条日志记录操作人、时间、内容
- 支持按设备或工单查询完整日志

### 5. 设备可用率统计
- 整体设备可用率计算
- 按区域维度统计可用率
- 支持区域筛选查看数据
